Ruby: Arrays and Hashes

October 24th, 2014

Today seems to be a significant date... 1024 = 2 ** 10. At any rate, today I'm going to talk about use of arrays and hashes in Ruby!

Arrays and hashes are both collections of data, in which indices are accessible with a key. Arrays are indexed by integers (starting with 0), while indices of hashes can be of any data type.

A hash stores key-value pairs and can be used for programs like phonebooks; you would look up a name (a string) to retrieve its associated phone number (another string). In this case, it would make sense to use a string to represent the data's index, rather than an integer. Looking for, say, the third phone number in the collection is not necessarily of interest to the user, hence integers would not be its ideal identification.

An array, on the other hand, is useful for creating data structures in which order is of importance, such as a list. Using an integer to represent a certain value would make more sense in ordering than, for example, a string. You may not have a data type that can better represent order than an integer (ie. the third item to do today vs. the yellow item to do).
