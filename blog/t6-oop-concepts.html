Variable Scope in Ruby

November 16th, 2014

Ruby allows for many types of variables, each with their own scope to allow flexible modeling of classes and objects. Here are a few that may be useful!

Global variables

Global variables are denoted with a $ sign, and can be accessed across classes and files.

Class variables

Class variables are denoted with two @ signs ("@@") and can be accessed from any instance of a class. However, if a class variable is accessed and reassigned from an instance, the value of the variable will change for every instance of the class. Class variables may be assigned outside a class definition, but it will run a warning.

Instance variables

Instance variables are denoted with a single @ sign. Much like class variables, they are accessible within a class and from any instance of the class; however, a reassignment of an instance variable will be unique to that particular instance.

Local variables

Local variables are accessible only within a method. They are also inaccessible across files.
