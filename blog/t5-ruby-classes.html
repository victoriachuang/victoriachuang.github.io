Ruby Classes

November 6th, 2014

Every object in Ruby belongs to a certain class (for example, strings belong to the String class). Classes determine an object's functionality and how you can manipulate data, so they're a convenient way to create several objects that you want to have similar behaviors (ie. have the same methods).

Methods

Built-in classes, such as arrays, strings, and integers, come with methods that can be called on instances of those classes. For example, in array1 = Array.new, array1 becomes a new instance of the array class and inherits all of its properties--therefore, you could call any array method on array1. You can also write your own instance and class methods, and even methods that only apply to one instance (singleton methods), to differentiate between objects and modify behavior.

Initializing class objects

Every class object is initialized with some certain properties. For example, if you were to make a new instance of class Car, you might initialize it with make, year, and color properties.

class Car
 def initialize(make, year, color)
  @make = make
  @year = year
  @color = color
 end
end

Notice that I'm setting the parameters to their equivalent with an @ sign; this allows me to access the variable in other methods in the class!
