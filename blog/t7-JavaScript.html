Learning JavaScript

November 20th, 2014

This week at Dev Bootcamp was focused on learning JavaScript. Before starting DBC, I had already completed the JS course on Codecademy, so I felt like I already had a decent grasp of the language and syntax. For the most part, I didn't think this week was too much of a challenge; I think getting past the Ruby hurdles made this week feel like a breeze!

Doing this week's challenges has made me reflect on how I learn best. I was really happy that I did the Codecademy track, because you code every step of the way, as opposed to reading ten pages and then coding one line. JavaScript is also the fourth language that I've learned, and a lot of parallels and similarities are starting to appear. As opposed to just learning a language, my approach now is to learn concepts. I feel like this is a better use of my time, as opposed to reading a book cover to cover, and re-reading about mathetmatical operators and why loops are useful.

Looking at the way loops differ in Ruby and JavaScript (good segue, right?), I feel like the syntax in Ruby is a lot more flexible. There are so many ways you can loop in Ruby:

x.times do
while
until
for
x.each do

For the most part, Ruby's syntax is very conversational; you type what you want to do, and that's the name of the method!

There are a lot of ways to loop in JS as well, but I feel like the syntax more closely mimics that of C. In JavaScript, one way to loop is (using printing 1 to 10 as an example):

for (var i = 1; i < 11; i++) {
 console.log(i);
}

The first line, following the for, there is a statement that includes an initialization, condition, and increments/decrements. The initialization, var i = 1, declares a variable i and assigns it a value of 1. The condition, i < 11, is checked every time the loop is performed; if the condition is true, then the loop will be executed. The last part of the statement, the increments/decrements, is i++; this ensures that i's value changes so that you don't end up in an infinite loop. i++ will add 1 to i each time the loop is executed--and once i is no longer less than 11, the loop will break.

I actually learned C way before I learned Ruby, so this syntax for constructing a loop is not foreign to me; plus, I think it teaches some concepts that are essential to computer science. Speaking of which, I think the Harvard CS50 and Stanford CS101 courses were immensely helpful in me picking up DBC material quickly and effectively; I would highly recommend it to anyone who wants supplementary material to watch; these instructors are really wonderful at explaining themselves. (I prefer the Stanford course, but Harvard's teaches C!).
